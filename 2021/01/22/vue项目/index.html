vue全家桶电商管理系统项目

登录/退出功能

登录业务流程

1.在登录页面输入用户名和密码
2.调用后台接口验证
3.通过验证后，根据后台的响应调到项目主页

登录业务的相关技术点

1.http是无状态的
2.通过cookie在客户端记录状态（不存在跨域问题）
3.通过session在服务器端记录状态（不存在跨域问题）
4.通过token方式维持状态（存在跨域问题）

token原理分析

（客户端）登录页面输入用户名和密码进行登录
（服务器）服务器端验证通过之后生成该用户的token并返回
（客户端）客户端存储token值
（客户端）后续所有的请求都携带该token发送请求给服务器端
（服务器）服务器验证token是否通过

如何进行表单验证

Form 组件提供了表单验证的功能，只需要在el-form通过 rules 属性传入约定的验证规则，并将 el-form-Item 的 prop 属性设置为需校验的字段名即可

如何重置表单

element-ui 对表单提供了如下方法 resetFields()，我们只需要获取表单对象数据即可重置我们的表单
直接在表单处el-form处添加ref属性比如LoginFormRef,在重置按钮@click绑定一个事件resetLoginForm，在methods方法里

登录预验证

在我们点击登录按钮，不应该直接发起网络请求，而是应该对表单进行预校验，检验成功才会发送请求。也就是拿到表单的引用对象也就是this.$refs.loginFormRef,在登陆按钮那绑定一个事件`@click=”login”,validate函数参数为一个回调函数。该回调函数会在校验结束后被调用，并传入两个参数：是否校验成功和未通过校验的字段。若不传入回调函数，则会返回一个 promise

完善登录之后的操作

1.将登录成功之后的token，保存到客户端的sessiontorage中
1.1项目中除了登录之后的API接口，必须在登录之后才能访问
1.2 token只应在当前网站打开期间生效，所以token保存在sessionStorage中

2.通过编程式导航跳转到后台主页，路由地址是/home

路由导航守卫控制权限

如果用户没有登录 但是直接通过URL访问特定页面 需要重新导航到登录页面。

这里可能会有疑问，我们需要验证 token的正确性吗？
答案是不需要，因为前端只需要管要不要给用户呈现这个页面，而我们调用api时，后端会对我们的 token 值进行校验，前端不需要验证。

退出功能

基于 token 的方式实现退出比较简单，只需要销毁本地的 token 即可。这样，后续的请求就不会携带 token ，必须重新登录生成一个新的 token 之后才可以访问页面。
